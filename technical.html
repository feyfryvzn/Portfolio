Recipe-Based Auto Stock Deduction

Sistem menerapkan prinsip Bill of Materials (BOM) yang diadaptasi ke industri kuliner. Setiap produk cookies terhubung ke tabel recipes (junction table many-to-many) yang memetakan kebutuhan bahan baku per unit produk.

DB::beginTransaction();
1. INSERT sale record
2. FOREACH recipe WHERE product_id = sold_product:
   deduction = qty_sold × recipe.quantity_needed
   IF raw_material.stock < deduction → ROLLBACK
   ELSE → UPDATE raw_materials SET stock -= deduction
3. CHECK: IF stock <= minimum_threshold
   → DISPATCH WhatsAppAlertJob (async)
DB::commit();

Seluruh operasi dibungkus dalam DB::transaction() dengan Pessimistic Locking untuk menjamin ACID Compliance dan mencegah race condition.

Smart Scheduling — Collision Prevention Algorithm

Algoritma Temporal Slot Validation memastikan tidak ada dua reservasi yang menggunakan teknisi yang sama pada waktu yang overlap.

1. Hitung end_time = requested_time + service.duration
2. Query overlap detection:
   SELECT COUNT(*) FROM reservations
   WHERE technician_id = ? AND date = ?
   AND status NOT IN ('CANCELLED','COMPLETED')
   AND (start_time < calc_end AND end_time > req_start)
3. IF count > 0 → REJECT + suggest alternatives
   ELSE → INSERT + DISPATCH confirmation

Kondisi overlap: new_start < existing_end AND new_end > existing_start — menangkap semua kemungkinan tumpang tindih baik sebagian maupun seluruhnya.

Arsitektur Notifikasi Terpadu

WhatsApp API diimplementasikan sebagai shared service yang digunakan lintas modul (Inventaris & Reservasi) melalui Laravel Queue.

1. Event Trigger

Sistem mendeteksi kondisi kritis (stok rendah / reservasi baru)

2. Queue Dispatch

Job dikirim ke Redis/Database queue secara asynchronous

3. Worker Process

Queue worker mengambil job dan mengirim HTTP request ke WhatsApp Gateway

4. Delivery & Retry

Jika gagal, job di-retry hingga 3x dengan exponential backoff

Detection Pipeline

INPUT (Image/Video/Webcam)
  → Preprocessing (Resize 640×640, Normalize, BGR→RGB)
  → YOLOv8 Inference (Single-Pass Detection)
  → Postprocessing (NMS Filter, Confidence ≥ 0.5)
  → OUTPUT: Annotated Image + JSON [{class, conf, bbox}]

Vehicle Detection → Toll Tariff Conversion

Model YOLOv8 mengklasifikasikan kendaraan ke dalam 5 golongan tol. Output deteksi (class_id) di-mapping ke tabel tarif untuk menghasilkan perhitungan biaya tol otomatis secara real-time.

class_id → CLASS_MAP["Golongan III"] → TARIFF[16500]

Output JSON:
{ "class": "Golongan III", "confidence": 0.92,
  "tariff": 16500, "timestamp": "2026-05-04T10:30:15" }

Strategi Indexing untuk High-Volume Data

Pada dataset ribuan SKU sparepart, Database Indexing diterapkan secara selektif untuk mengubah Full Table Scan menjadi Index Seek.

TANPA INDEX: 15,000 rows examined → ~120ms
DENGAN INDEX: 1 row examined → ~0.5ms

-- Composite Index untuk multi-kondisi query
CREATE INDEX idx_sales_customer_date
  ON sales_orders(customer_id, order_date);

Referential Integrity

Foreign Key Constraints dengan strategi ON DELETE RESTRICT mencegah penghapusan sparepart yang masih memiliki transaksi aktif, menjaga konsistensi data relasional.
